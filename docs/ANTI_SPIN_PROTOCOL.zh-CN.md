# Lychee AlphaDesk 防原地打转协议

本协议用于约束后续开发，防止项目只是在菜单、队列、文档和流程包装上循环，而没有向“最终投资工作台”前进。

## 1. 当前北极星

Lychee AlphaDesk 的第一阶段目标不是成为看 K 线工具，也不是交易机器人，而是：

- 帮初学者从美股、港股、A 股的大量信息中发现值得研究的线索。
- 把新闻、行情、公告、财报、行业链条和 LLM 分析组织成可审计研究流程。
- 每一步都给出可执行的下一步研究动作，但不提供买入、卖出、持有、仓位、目标价或收益预期。

## 2. 防打转门禁

每个阶段开始前，必须先写清楚：

1. **本阶段怎样让目标更真**  
   必须说明它提升的是“发现线索、核验证据、补齐数据、形成研究判断”中的哪一个能力。

2. **本阶段不做什么**  
   如果只是新增菜单、重排文案、包装已有队列，默认判定为低价值，除非它直接解除用户无法继续研究的阻塞。

3. **真实输出验收**  
   不能只用单元测试证明完成。必须运行至少一个真实 CLI/TUI 路径，检查输出是否比上一版更接近投资工作台。

4. **退出条件**  
   阶段完成时必须有一个可复用命令或 TUI 入口，且输出必须能回答“我现在应该研究什么，为什么，证据在哪里，下一步怎么验证”。

5. **队列消耗证明**
   `下一步行动队列` 中的可执行动作完成后，必须写入可审计 artifact/SQLite 记录，并让已完成的同一待办从后续队列中消失或转成明确的人工/数据源缺口。`no-data` 必须写入可审计 cooldown/缺口记录，并在保质期内让队列跳到下一个可执行研究动作；成功补到数据的动作必须转成下一步研究动作，下一步完成后也必须从队列消失；`failed` 必须明确停止或转人工处理。`no-data` 和 `failed` 不能打印下一步核验命令，也不能伪装成已完成。

6. **命令连续性证明**
   工作台、下一步行动队列、证据板和复核结果中打印的下一步命令必须能复制后直接执行，并回到同一个研究任务。凡是从扩大扫描范围生成的任务，后续 `research run`、`research verify`、`research review`、`research memo` 和 `research evidence-review` 命令必须继承必要的 `--limit`；用户明确输入直接证券代码时，直接 symbol 候选优先于把同一代码当作代理标的匹配到主题任务。

## 3. 打转信号

出现以下情况时，必须停止当前方向并重新选择核心能力切片：

- 连续两个提交只改变菜单、表格、文案、队列展示或文档。
- 真实输出仍然只围绕已知热门标的，例如 QQQ、NVDA、TSLA，而没有解释为什么此刻值得研究。
- 新增功能只能告诉用户“缺少数据”，却不能自动寻找、计算、映射或验证任何新证据。
- LLM 输出仍然是泛泛结论，不能形成可核验的研究假设、反证条件或下一批数据请求。
- 用户仍然需要自己知道股票代码，系统才开始工作。
- 下钻核验仍然只是证据列表或教学说明，没有把当前信号、反向压力、证据缺口和下一步研究动作翻译成工作台读数。
- 队列或证据板打印的命令复制执行后找不到同一研究任务，或跳到同一 symbol 的代理主题而不是直接候选。

## 4. 后续优先级

后续开发按以下顺序推进：

1. **机会雷达**：从市场新闻、行情和本地缓存中发现异常线索。
2. **主题到标的映射**：把新闻主题映射到行业、供应链、ETF、指数和可交易标的。
3. **证据评分**：把支持、反向、待判定和缺口证据转成研究优先级。
4. **LLM 二阶段分析**：只在结构化证据准备好之后调用 LLM，避免一次性塞入原始杂讯。
5. **TUI 执行闭环**：让用户可以从机会雷达进入研究、补数据、复核证据、生成备忘录。

## 5. 阶段验收模板

每次提交前必须自查：

- 目标推进：本阶段让哪个核心能力更强？
- 真实命令：运行了哪个 `lychee ...` 命令？
- 输出检查：输出是否更像投资工作台，而不是教程或任务清单？
- 命令连续性：复制输出里的下一步命令是否能回到同一任务？
- 证据强度：是否使用了真实缓存、真实 provider 返回或可审计本地数据？
- 剩余风险：下一阶段最该补的能力是什么？
